# 辺境の冠（仮称）

> **Cities: Skylines系の都市開発に、四季・大量人口・領土拡張・政治・外交・独立戦争・旧王国打倒を重ねた、2Dドット絵の王国建設シミュレーション。**

- 状態: 企画・仕様策定
- README版: 0.3.0
- 想定プラットフォーム: PCブラウザ
- 配布形態: 原則 `index.html` 1ファイル
- 起動方法: `index.html` をダブルクリック
- サーバー: 不要
- 通信: 不要
- 推奨ブラウザ: Chrome / Edge
- 描画: Canvas 2D
- セーブ: ブラウザ内保存 + JSON書き出し
- 外部ファイル: BGM・効果音のみ許容

# 1. ゲーム概要

プレイヤーは、旧王国の支配がほとんど届いていない辺境領の管理者としてゲームを開始する。

土地は法的には旧王国の一部だが、中央政府の役人も軍隊もほとんど存在しない。プレイヤーは道路、住宅区、農地、工房、市場、井戸、倉庫、役所、城壁などを整備し、小さな開拓拠点を都市へ成長させていく。

やがて周囲の村や土地を取り込み、複数都市を束ねた自治領へ発展する。しかし、発展すればするほど旧王国から危険視される。

- 税や物資を要求される
- 兵士を王国軍へ送るよう命じられる
- 独自軍や城塞を制限される
- 外国との外交を禁じられる
- 監察官や王国軍を送り込まれる
- 地方貴族や住民を扇動される
- 最終的には反逆者として討伐される

プレイヤーは旧王国へ忠誠を示しながら力を蓄えることも、他国や地方貴族と手を組んで独立することも、旧王国の内乱や王位継承争いへ介入して新たな王になることもできる。

**都市を作った結果として国が生まれ、国を育てた結果として歴史が変わるゲームを目指す。**

---

# 2. 開発上の最重要方針

1. 小さな辺境集落が巨大国家へ成長する実感を出す
2. 都市を自分で設計・発展させる楽しさを中心に置く
3. ステージごとにゲームのミソを変える
4. 人口の多さは妥協しない
5. できるだけ低スペックPCでも動作させる
6. 原則として `index.html` 1ファイルで完結させる
7. サーバーを立てず、ダブルクリックだけで遊べるようにする
8. 外部ライブラリ、CDN、API、常時通信へ依存しない

# 4. ゲームのステージ

同じことを規模だけ大きくして繰り返さない。

| 段階 | 主役になる要素 | その段階のミソ |
|---|---|---|
| 開拓拠点 | 生存 | 水・食料・燃料・住居を確保する |
| 集落 | 都市建設 | 道路、雇用、物流、衛生を整える |
| 辺境領 | 複数拠点 | 地形ごとの生産拠点を街道でつなぐ |
| 自治領 | 政治 | 発展しつつ旧王国に警戒されない |
| 半独立国家 | 勢力工作 | 貴族、派閥、他国を味方につける |
| 独立戦争 | 補給と崩し | 王国軍と正面衝突せず国家を崩す |
| 独立王国 | 再建 | 戦後の混乱を収め正統性を作る |
| 王位争奪 | 大戦略 | 王位請求、革命、征服、経済支配 |
| 新王朝 | 帝国統治 | 広大な国家を崩壊させず維持する |

# 5. 時間と四季

## 5.1 基本時間

1年を12か月とし、各季節を3か月とする。

| 季節 | 月 | 主なテーマ |
|---|---:|---|
| 春 | 1〜3月 | 作付け、雪解け、洪水、移民 |
| 夏 | 4〜6月 | 建設、成長、火災、疫病、干ばつ |
| 秋 | 7〜9月 | 収穫、交易、備蓄、徴税 |
| 冬 | 10〜12月 | 食料、燃料、寒さ、補給、移動困難 |

時間速度:

- 一時停止
- 1倍速
- 2倍速
- 4倍速
- デバッグ用高速進行

重大イベント発生時は自動停止できる。

## 5.2 春

- 作付け
- 雪解け
- 河川増水
- 洪水
- 道路のぬかるみ
- 移民増加
- 建設再開
- 冬季損傷の修理

春に農民を徴兵や建設へ回しすぎると、秋の収穫量が減る。

## 5.3 夏

- 建設速度上昇
- 作物成長
- 交易活発化
- 人口増加
- 水不足
- 火災
- 疫病
- 食料腐敗
- 森林火災

都市の成長に井戸、衛生、消防、食料流通が追いつくかが重要になる。

## 5.4 秋

- 農作物収穫
- 食料価格低下
- 荷車と商人増加
- 税収増加
- 冬支度
- 軍需備蓄
- 旧王国の徴税

収穫物は備蓄、売却、献上、軍用、外交援助へ振り分ける。

## 5.5 冬

- 農業停止
- 食料消費増加
- 燃料消費
- 建設速度低下
- 商人減少
- 道路移動低下
- 河川凍結
- 凍死
- 病気
- 軍補給悪化

冬季予測を表示する。

```text
食料備蓄: 4.2か月分
燃料備蓄: 2.8か月分
住宅不足: 74世帯
凍死危険: 中
```

## 5.6 地域気候

- 北部: 冬が長く厳しい
- 南部: 冬は軽いが干ばつが多い
- 山岳: 雪が早く春が遅い
- 海岸: 温暖だが嵐が多い
- 湿地: 洪水と疫病が多い
- 乾燥地: 雨季と乾季の影響が強い

### 12.2.1 初期プロトタイプの労働管理

個人ごとの年齢・職歴はまだ持たず、都市単位と施設単位の集団データとして処理する。

```text
総人口
├─ 労働可能人口（現在は総人口の60%）
│  ├─ 就業者
│  └─ 失業者
└─ 扶養人口
```

各施設は必要労働者数と配属優先度を持ち、労働可能人口を優先度順に自動配属する。

```text
最優先: 井戸、燃料
高: 食料生産、パン焼き窯
中: 製粉所、石材、鉱石
低: 鍛冶場、将来の高度産業
```

畑と採集所は冬季の必要労働者数を減らし、余った労働者を他施設へ自動的に再配属する。

施設の実生産量は次を整数計算した値とする。

```text
実生産量
= 季節別基礎生産量
× 労働者充足率
× 原料充足率
× 道路接続率
```

共有在庫方式は維持し、実物流を導入するまでは労働者と原料の充足率で施設稼働を表現する。
調査カードには、配属人数、必要人数、労働充足率、原料充足率、総合稼働率、基礎生産量、実生産量、停止理由を表示する。

# 13. 物流と交通

## 13.1 物流

- 農村から都市へ食料
- 森林から工房へ木材
- 鉱山から鍛冶屋へ鉄
- 工房から市場へ製品
- 倉庫から軍へ補給
- 港から他国へ輸出

国全体に資源があっても、道路や倉庫が悪ければ必要な都市へ届かない。

### 13.1.1 初期プロトタイプの共有在庫と保管上限

実物流を導入する前段階として、生産物は道路接続された施設から都市の共有在庫へ直接入る。
ただし共有在庫は無制限にせず、資源ごとに都市の基礎保管容量を持つ。

| 資源 | 基礎容量 | 資材置き場1棟の最大追加容量 |
|---|---:|---:|
| 食料 | 15,000 | 5,000 |
| 燃料 | 6,000 | 3,000 |
| 木材 | 800 | 1,000 |
| 穀物 | 400 | 1,500 |
| 小麦粉 | 200 | 800 |
| 石材 | 400 | 1,200 |
| 鉱石 | 200 | 600 |
| 道具 | 100 | 300 |

金は保管上限の対象外とする。
資材置き場は4人の倉庫管理労働者を必要とし、追加容量は労働者充足率と道路接続率を掛けた整数値になる。
未接続、労働者不足、撤去によって容量が下がった場合も、上限を超えた在庫は廃棄して資源別の直近値と累計値を記録する。

```text
実保管容量
= 都市の基礎保管容量
+ Σ（資材置き場の基礎追加容量 × 労働者充足率 × 道路接続率）
```

日次処理では、生産施設と加工施設の入出力、住民消費をすべて処理した後に上限を適用する。
これにより、同じ日に生産した原料を加工施設が利用できる一方、最終的にあふれた在庫だけが廃棄される。
冬季予測も準備期間と冬季開始時の保管上限を反映する。

この段階では荷物の予約、荷車ごとの積載、施設別在庫、渋滞は扱わない。
表示住民と荷物は共有在庫計算の視覚化に留め、実物流は市場と配送効率を導入してから詳細化する。

## 13.2 軽量処理

- 道路区間ごとに交通量を集計
- 混雑度を一定間隔で更新
- 主要物流経路をキャッシュ
- 道路変更時だけ再探索
- 表示荷車は交通量を表現するサンプル

# 35. 完成時の理想

最初は数十人しかいなかった開拓地へ獣道が敷かれ、井戸と畑が作られ、冬を越す。

やがて市場、倉庫、街道、鉱山町、農業村が生まれる。

人口が増え、都市が混雑し、病気や火災が起き、住民の生活と物流を改善する必要が生まれる。

発展した辺境領へ旧王国の徴税官が現れ、税と兵士を要求する。

プレイヤーは忠誠を装いながら軍備を整え、他国や地方貴族と秘密裏に交渉する。

独立戦争では、自分が建設した街道、倉庫、城壁都市、港が軍事と補給の基盤になる。

最終的には旧王国の王都を攻略するか、王位を継承するか、革命を起こすか、経済的に支配し、新たな王朝を作る。

そして新王朝の辺境では、かつての自分と同じような小さな勢力が成長を始める。

**都市を作り、季節を生き、国を広げ、王国を倒し、自分自身が歴史の一部になるゲームを目指す。**
